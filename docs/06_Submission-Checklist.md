# Submission Checklist

This is a checklist to ensure all required artifacts exist.

## Backlog & Sprint Plans
- [ ] Product vision
- [ ] Complete product backlog (≥5 stories) with acceptance criteria, priority, estimates
- [ ] Definition of Done
- [ ] Sprint 1 plan (2–3 selected stories)
- [ ] Sprint 2 plan (2–3 selected stories)

## Codebase
- [ ] GitHub repository link
- [ ] Commit history shows incremental delivery

## CI/CD Evidence
- [ ] Workflow file in `.github/workflows/`
- [ ] Screenshot(s) or log snippet(s) of successful run
- [ ] Screenshot(s) or log snippet(s) of a failed run (optional but helpful)

## Testing Evidence
- [ ] Test files included in repo
- [ ] Screenshot(s) or log output of passing tests

## Sprint Reviews
- [ ] Sprint 1 review doc (what was delivered + evidence)
- [ ] Sprint 2 review doc (what was delivered + evidence)

## Retrospectives
- [ ] Sprint 1 retro (≥2 concrete improvements)
- [ ] Sprint 2 retro + final reflection
